# Local RAG without Docker: Open WebUI AppImage + Ollama + ChromaDB

This setup runs everything natively on your machine with **one startup script**:

- **Ollama** provides local chat models and embedding models.
- **ChromaDB** stores document embeddings for RAG.
- **Open WebUI AppImage** provides the browser frontend.

Open WebUI is launched with environment variables that point it to local Ollama and local ChromaDB.

## Prerequisites

Install these on the host machine:

- Python 3.11+ recommended
- Ollama: <https://ollama.com/download>
- Your existing Open WebUI AppImage

Verify Ollama is available:

```bash
ollama --version
```

## 1. Configure `.env`

Create `.env` if it does not exist:

```bash
cp .env.example .env
```

Edit `.env` and set your AppImage path:

```env
OPEN_WEBUI_APPIMAGE=/home/abhinav/Downloads/Open-WebUI.AppImage
```

Also replace `WEBUI_SECRET_KEY` with a long random value. Example:

```bash
openssl rand -hex 32
```

Useful model settings in `.env`:

```env
CHAT_MODEL=llama3.2:3b
RAG_EMBEDDING_MODEL=nomic-embed-text
```

Important:

- `CHAT_MODEL` must be a chat-capable model.
- `RAG_EMBEDDING_MODEL` is for embeddings only.
- Do **not** select `nomic-embed-text` as the chat model in Open WebUI.

## 2. Start everything with one script

Run:

```bash
sh scripts/start_all.sh
```

The script will:

1. Create `.venv` if missing.
2. Install `chromadb` if missing.
3. Start Ollama if it is not already running.
4. Pull the chat model from `CHAT_MODEL` if missing.
5. Pull the embedding model from `RAG_EMBEDDING_MODEL` if missing.
6. Start ChromaDB.
7. Start your Open WebUI AppImage with the RAG environment variables.

Open WebUI is usually available at:

```text
http://127.0.0.1:8080
```

Keep the script running. Press `Ctrl-C` in that terminal to stop services started by the script.

Logs are written to:

```text
./logs/ollama.log
./logs/chromadb.log
./logs/open-webui.log
```

## RAG configuration

The `.env.example` file configures Open WebUI with:

```env
OLLAMA_BASE_URLS=http://127.0.0.1:11434
VECTOR_DB=chroma
CHROMA_HTTP_HOST=127.0.0.1
CHROMA_HTTP_PORT=8000
RAG_EMBEDDING_ENGINE=ollama
RAG_EMBEDDING_MODEL=nomic-embed-text
```

This means:

- Chat completions go to local Ollama.
- Document/query embeddings are generated by Ollama using `nomic-embed-text`.
- Embeddings are stored and searched in the local ChromaDB server.

## Use RAG in Open WebUI

1. Open Open WebUI in your browser.
2. Go to **Workspace** / **Knowledge**.
3. Upload your documents.
4. Create or select a knowledge collection.
5. Start a chat with a chat-capable Ollama model such as `llama3.2:3b`.
6. Do **not** select `nomic-embed-text` as the chat model; it is only for embeddings.
7. Attach/select the knowledge collection and ask questions about your documents.

## Manual startup, only if needed

The recommended path is `sh scripts/start_all.sh`. If you need to debug services separately, use these commands.

Start Ollama:

```bash
ollama serve
```

Start ChromaDB:

```bash
. .venv/bin/activate
./scripts/start_chromadb.sh
```

Start Open WebUI AppImage:

```bash
./scripts/start_openwebui_appimage.sh
```

## Troubleshooting

### `nomic-embed-text:latest` does not support chat

This is normal. `nomic-embed-text` is an embedding model, not a chat model.

Use it only for RAG embeddings via:

```env
RAG_EMBEDDING_ENGINE=ollama
RAG_EMBEDDING_MODEL=nomic-embed-text
```

For chat, select a chat-capable model in Open WebUI, for example:

```bash
ollama pull llama3.2:3b
```

Then select `llama3.2:3b`, not `nomic-embed-text`, in the chat model dropdown.

### Open WebUI cannot connect to Ollama

Check Ollama is running:

```bash
curl http://127.0.0.1:11434/api/tags
```

If that fails, start everything with:

```bash
sh scripts/start_all.sh
```

### ChromaDB is not reachable

Check:

```bash
curl http://127.0.0.1:8000/api/v1/heartbeat
```

If that fails, check:

```bash
cat logs/chromadb.log
```

### RAG upload or search fails

Make sure the embedding model exists locally:

```bash
ollama list
```

If `nomic-embed-text` is missing, the one-script startup should pull it automatically. You can also pull it manually:

```bash
ollama pull nomic-embed-text
```

Then restart Open WebUI through:

```bash
sh scripts/start_all.sh
```

### AppImage path error

Set `OPEN_WEBUI_APPIMAGE` in `.env` to the full path of your AppImage:

```env
OPEN_WEBUI_APPIMAGE=/home/abhinav/Downloads/Open-WebUI.AppImage
```

The startup script will try to make it executable automatically if needed.

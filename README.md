# Local RAG: Streamlit UI + Ollama + ChromaDB

This setup runs everything natively on your machine with **one startup script**:

- **Ollama** provides local chat models and embedding models.
- **ChromaDB** stores document embeddings for RAG.
- **Streamlit** provides the browser frontend.

## Prerequisites

Install these on the host machine:

- Python 3.11+ recommended
- Ollama: <https://ollama.com/download>
- Streamlit

Verify Ollama is available:

```bash
ollama --version
```

## 1. Configure `.env`

Create `.env` if it does not exist:

```bash
cp .env.example .env
```

Useful model settings in `.env`:

```env
CHAT_MODEL=llama3.2:3b
RAG_EMBEDDING_MODEL=nomic-embed-text
```

Important:

- `CHAT_MODEL` must be a chat-capable model.
- `RAG_EMBEDDING_MODEL` is for embeddings only.
- Do **not** select `nomic-embed-text` as the chat model in Streamlit.

## 2. Start everything with one script

Run:

```bash
./scripts/start_all.sh
```

The script will:

1. Create `.venv` if missing.
2. Install `chromadb` if missing.
3. Start Ollama if it is not already running.
4. Pull the chat model from `CHAT_MODEL` if missing.
5. Pull the embedding model from `RAG_EMBEDDING_MODEL` if missing.
6. Start ChromaDB.
7. Start Streamlit with the RAG environment variables.

Streamlit is usually available at:

```text
http://127.0.0.1:8501
```

Keep the script running. Press `Ctrl-C` in that terminal to stop services started by the script.

Logs are written to:

```text
./logs/ollama.log
./logs/chromadb.log
```

## RAG configuration

The `.env.example` file configures with:

```env
OLLAMA_BASE_URLS=http://127.0.0.1:11434
VECTOR_DB=chroma
CHROMA_HTTP_HOST=127.0.0.1
CHROMA_HTTP_PORT=8000
RAG_EMBEDDING_ENGINE=ollama
RAG_EMBEDDING_MODEL=nomic-embed-text
```

This means:

- Chat completions go to local Ollama.
- Document/query embeddings are generated by Ollama using `nomic-embed-text`.
- Embeddings are stored and searched in the local ChromaDB server.

## Manual startup, only if needed

The recommended path is `./scripts/start_all.sh`. If you need to debug services separately, use these commands.

Start Ollama:

```bash
ollama serve
```

Start ChromaDB:

```bash
. .venv/bin/activate
./scripts/start_chromadb.sh
```

## Troubleshooting

### `nomic-embed-text:latest` does not support chat

This is normal. `nomic-embed-text` is an embedding model, not a chat model.

Use it only for RAG embeddings via:

```env
RAG_EMBEDDING_ENGINE=ollama
RAG_EMBEDDING_MODEL=nomic-embed-text
```

For chat, select a chat-capable model in WebUI, for example:

```bash
ollama pull llama3.2:3b
```

Then select `llama3.2:3b`, not `nomic-embed-text`, in the chat model dropdown.

### Streamlit cannot connect to Ollama

Check Ollama is running:

```bash
curl http://127.0.0.1:11434/api/tags
```

If that fails, start everything with:

```bash
./scripts/start_all.sh
```

### ChromaDB is not reachable

Check:

```bash
curl http://127.0.0.1:8000/api/v1/heartbeat
```

If that fails, check:

```bash
cat logs/chromadb.log
```

### RAG upload or search fails

Make sure the embedding model exists locally:

```bash
ollama list
```

If `nomic-embed-text` is missing, the one-script startup should pull it automatically. You can also pull it manually:

```bash
ollama pull nomic-embed-text
```

Then restart  Streamlit through:

```bash
./scripts/start_all.sh
```

The startup script will try to make it executable automatically if needed.

## Docker (Ollama + ChromaDB + Streamlit)

This runs the full stack in containers. The Streamlit container uses the same
`./scripts/start_all.sh` entrypoint as local development, with Ollama and ChromaDB
provided by sibling containers.

Models are **downloaded on first start** into a Docker volume (`ollama_data`), so later
restarts reuse them.

### Prerequisites

- Docker Engine + Docker Compose v2
- Enough disk space for models (~2–3 GB for `llama3.2:3b` + `nomic-embed-text`)
- Optional: NVIDIA GPU + [nvidia-container-toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html) for faster inference

### Quick start

From the project root:

```bash
docker compose up --build
```

First startup can take several minutes while Ollama pulls models. Watch progress:

```bash
docker compose logs -f ollama
```

Open the app:

```text
http://127.0.0.1:8501
```

### Services

| Service   | Port  | Role                                      |
|-----------|-------|-------------------------------------------|
| `app`     | 8501  | Streamlit RAG UI                          |
| `ollama`  | 11434 | LLM + embedding server                    |
| `chromadb`| 8000  | Vector store for document search          |

### Customize models

Set variables in `.env` (or export before `docker compose up`):

```env
CHAT_MODEL=llama3.2:3b
RAG_EMBEDDING_MODEL=nomic-embed-text
STREAMLIT_PORT=8501
```

On next start, Ollama pulls any missing models automatically.

### GPU support

Uncomment the `deploy.resources.reservations.devices` block under `ollama` in
`docker-compose.yml`, then restart:

```bash
docker compose down
docker compose up --build
```

### Persisted data

Docker volumes keep data across restarts:

- `ollama_data` — downloaded models
- `chroma_data` — embedded document chunks
- `app_data` — chat history, system prompt, analytics

Remove everything (including models):

```bash
docker compose down -v
```

### Stop

```bash
docker compose down
```

### How models are bundled

Models are **not baked into the image** (that would make images very large). Instead,
`scripts/docker-ollama-entrypoint.sh`:

1. Starts `ollama serve`
2. Pulls `CHAT_MODEL` and `RAG_EMBEDDING_MODEL` if not already in the volume
3. Keeps the server running

To pre-seed models on a machine without waiting during compose startup, you can also run:

```bash
docker compose up -d ollama
docker exec -it rag-ollama ollama pull llama3.2:3b
docker exec -it rag-ollama ollama pull nomic-embed-text
docker compose up -d
```
